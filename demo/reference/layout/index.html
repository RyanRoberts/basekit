Layout Grid

Basekit's grid system is a faux-grid built on CSS inline-block and uses the data attribute rather than classes.

Once white space has been removed, inline-block provides the flexibility and robustness that floats do not. Inline-block items can be vertically aligned and flow more consistently.

The data attribute allows us to almost fully separate the layout styles from other CSS and classes.

The Grid

By default BaseKit is based on the common 12 column layout, this provides great flexibility and ease of use. However you can create layouts from as many or as few columns as you wish.

Unlike Bootstrap and some other systems, there are no "tiers". If you want a four column grid at small screen size simply fake it by using three columns for each small screen column.

Padding

Background colour indicated where the padding has been applied.

Padding Sizes

There are three sizes of padding, the default value and small and large values. These can be configured via the BaseGrid variables.

Small padding is particularly useful if you want to combine the padding/gutters of a container as well as the child while maintaining the standard padding/gutter size.
