Typography

Basekit offers various font classes for very targetted styling.

Fonts can be fluid or fixes. Fluid fonts will adjust in size between a specified min and max size limits depending on the screen size. Fixed fonts will remain the same size regardless of the screen size. Fluid fonts are ideal for headings short blocks of text while fixed fonts are best suited to long blocks of text.

In addition to the font style classes, a rich text content wrapper class is also available that scopes and styles content output from rich text editor fields.

Font Sizes

In Basekit all headings are set to the same size by default, since it is not necessarily true that you would want a large h1 and a small h4. Sometimes you might want them to be the same size, othertimes it may be have visual weight that does not consist with semantic weight.

Due to this, Basekit abstracts out font weights and styles to specific utility classes which must be applied to get the desired style. Otherwise all headings and text sizes will, for the most part, remain the same. This also has the added advantage of being able to apply consistent styles to any text elements, regardless of the markup.

Rich-text editor / User content

Rich-text content is, typically, content output by the Content Management System and written by the user in a Rich-Text Editor field. Here we tightly scope the content from the rest of the website content to reduce conflicts or the need to reset styles.

Link styles are handled by a separate link scope selector rather than the .rte class, this allows greater freedom when styling links.

The goal of the .rte scope class is to cover as many common situations as is sensible without creating too much unused CSS. Users should not have free reign with rich-text editors otherwise they can quickly cause mayhem.

Note for developers

The .rte class sets heading styles that will override some .font-* classes. Keep this in mind and try to use the class only where necessary and try to wrap only the required content.
