Colours

Primary and Secondary colour groups are often used for headings, links, button and background colours. Links and buttons may default to the primary colour with the secondary colour used on hover.

Often a tertiary colour option won't be provided. Tertiary colours are useful for active link states, buttons, and background colours.

Brand Colours

Primary and Secondary values will be defined in by brand guidelines or style guides provided by the designer or client. The lighter and darker values are not always the same and should be tweaked in the Color settings Sass file. These tings should be set for a balance of aesthetic appeal and utility, a dark that is too light or a light that is too dark won't be of any use when building the website.

The tertiary value is not always provided as part of the brand guidelines or style guide, in such a case talk to the designer responsible or find an appropriatly fitting grey.

Note for designers

You're not expected to provide the additional four light/dark values, only the three main Primary, Secondary and Tertiary. The four additional values are tweaked to best fit actual use throughout the site, often the best value is only found once the functional UI starts coming together.

Default Text Colour

For text colours the lightest you can go on a white background is #666 for AA compliancy or #555 for AAA compliancy. It is advisable to keep the base text colour well above AAA compliancy.

On a black background, light text can go as dark as 999 for AAA compliancy and 666 for AA compliancy. Again it is advised to stick with AAA level compliancy.

For body text on mid-range and different colours you must check with a contrast testing tool. For more information see the Accessibility page.
